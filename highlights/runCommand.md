# Compiling like a PRO

C++ can use tons of different flags to change how the compiler will assemble the program, some might help you debug your code to find errors that a simple compiling might not catch:

### Simplest compiling command
### <center> `g++ file.cpp`

---
### Advanced compiling command
### <center> `g++ -std=c++17 -O2 -Wall file.cpp -o file`

---

### Problems Mega-Finder command (only works in Linux)

### <center> `g++ -std=c++20 -DBRUTE -O2 -Wall -Wextra -Wconversion -Wfatal-errors -fsanitize=address,undefined file.cpp && ./a.out`

# Explaining Flags
- `-std=c++20`: The C++ version that g++ will use
- `-O2`: Optimization level **2**, higher optimizations might make the code behave differently
- `-Wall`: **Warning ALL**, will show all warnings that the program might have, and not only errors
- `-Wextra`: **Warning Extra**, shows extra warnings that `-Wall` won't show
- `-Wconversion`: Show warnings about type conversions (e.g. an int receives a double)
- `-Wfatal-errors`: Stops the error logs on the first error, so you can fix each individually
- `-fsanitize=address,undefined`: Injects verifiers in the code, showing problems related to the tags:
    - `address`: e.g. Access to out of bounds positions in a vector
    - `undefined`: Multiple behaviours such as overflows or dividing by 0
- `file.cpp`: Name of the file that will compile
- `&& .\a.out`: runs the executable generated by the line
